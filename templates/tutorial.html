Tutorial

AlgoJ is a python based judge system that you can use for testing your source code. The archive contains a list of algorithmic tasks which you can solve. All solutions are judged using Python3. This is a prototype system, so don't be surprised if you exeperience a bug. Use this sytem mainly for training, it's not used for official grading.

Registration

To access the archive, you first need to register. The registration is a simple process, you can use your university email or your personal one. Press the Sign up button in the top right corner in order to register.

Problem Archive

The problem archive contains a list of tasks available for solving. Each task is composed of a:

  1. Problem statement
  2. Input and output specification
  3. One or more examples

Submission

In order to solve a problem, you must write a function(s) that conforms to the specification of the problem. In most cases, the input and output will be specified in the statement, and the code template will already include the necessary code in order in read the data and write the results. You only have to implement the specified functions.

After submitting your code, go to the Submissions page in order to track your results.

Verdicts

Here is a list of possible verdicts that your solution can get:
  1. Accepted - it means that your code got the right answer for the test.
  2. Wrong Answer - it means your code got a wrong answer for a given test.
  3. Time Limit Exceeded - you code has passed over the time limit. Each problem has a specified time limit.
  4. Runtime Error - your solution exited with a non-zero code, or your solution contains non-valid python code. Usually you'll be able to see a small description in the details if this happens.
  5. Evaluation Error - this should not happen. It means something malfunctioned in the judge. Try sending your submission again, or contact the administrator.
In addition, each problem has memory limit which is not specified but is set to around 256 MB. {% endblock %}
